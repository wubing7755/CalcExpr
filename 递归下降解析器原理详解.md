# 递归下降解析器原理详解

## 一、为什么需要"递归下降"？

### 1.1 表达式可以表示为树

表达式 `9*8-1` 可以表示为一棵**语法树**：

```
          (-)
         /   \
        (*)   (1)
       /   \
      (9)   (8)
```

**树的特点：**
- 叶子节点是数字
- 内部节点是运算符
- 子树先计算（优先级高的运算符在低层）

### 1.2 递归下降的核心思想

**用函数调用来构建和遍历树**

```c
parseExpression()  → 生成 (-) 节点
    └─ parseTerm()  → 生成 (*) 节点
            ├─ parseUnary() → 返回 9
            └─ parseUnary() → 返回 8
```

每个语法规则对应一个函数，函数之间互相调用来"分解"表达式。

---

## 二、为什么需要分层（优先级）

### 2.1 运算符优先级问题

数学中，乘法比加法先计算：

```
9+8*1  ≠  (9+8)*1
  11    ≠    17
```

### 2.2 如何实现"先乘除后加减"

**方法：分层处理，每层处理一种优先级**

```
Expression (处理 + -)          ← 最低优先级，最后计算
    └─ Term (处理 * /)        ← 中等优先级，中间计算
            └─ Unary (处理负号) ← 较高优先级
                    └─ Primary (数字和括号) ← 最高优先级，最先计算
```

**执行顺序（从最内层开始）：**

```
1. Primary 读取数字 9, 8, 1
2. Unary 处理一元运算符（本例无）
3. Term 计算 9*8=72
4. Expression 计算 72-1=71
```

## 三、为什么用"循环"而不是"递归"处理运算符链？

### 3.1 问题：`9*8*2` 如何处理？

**错误思路（左递归，会无限递归）：**

```
Expression ::= Expression '*' Term  ← 永远以 Expression 开头，无限递归
```

**正确思路（使用循环）：**

```
Expression ::= Term { ('+' | '-') Term }
               ↑
               └─ 先获取左操作数
                              ↑↑↑↑↑↑
                              └─┘ │
                              循环处理后续操作数
```

### 3.2 循环处理的原理

```c
// 伪代码
double parseExpression() {
    // 第1步：获取第一个操作数
    left = parseTerm();

    // 第2步：循环处理后续运算符
    while (当前运算符是 + 或 -) {
        op = 当前运算符;        // 保存运算符
        parserNextToken();      // 消费运算符
        right = parseTerm();    // 获取右操作数
        left = applyOp(left, op, right);  // 计算并更新 left
    }

    return left;
}
```

### 3.3 图解 `9*8*2` 的处理流程

```
输入: 9*8*2

第1次循环:
  left = 9
  遇到 *, 保存 op=*, 消费 *
  right = 8
  left = 9*8 = 72

第2次循环:
  left = 72
  遇到 *, 保存 op=*, 消费 *
  right = 2
  left = 72*2 = 144

循环结束（遇到 END）
返回 144
```

## 四、括号的处理原理

### 4.1 括号为什么特殊？

因为括号**改变了优先级**：

```
(9+1)*2  中，9+1 必须先算
```

### 4.2 如何处理括号

**关键：遇到 `(` 时，递归调用解析器本身**

```c
double parsePrimary() {
    if (是数字) {
        return 数字值;
    }

    if (是 '(') {
        parserNextToken();              // 消费 '('
        value = parseExpression();       // ⚡ 递归！重新从 Expression 开始
        // 此时括号内的表达式会被完整解析
        parserNextToken();              // 消费 ')'
        return value;
    }
}
```

### 4.3 图解 `(9+1)*2`

```
输入: (9+1)*2

主流程 (Expression):
  └─ Term:
        ├─ Unary:
        │     └─ Primary: 遇到 '('
        │           ├─ 消费 '('
        │           ├─ 递归 parseExpression()
        │           │
        │           │  ⚡ 递归层：parseExpression()
        │           │    ├─ parseTerm() → 9
        │           │    ├─ 遇到 '+'
        │           │    ├─ parseTerm() → 1
        │           │    └─ 返回 9+1=10
        │           │
        │           ├─ 消费 ')'
        │           └─ 返回 10
        │
        ├─ 遇到 '*'
        └─ Unary → Primary: 返回 2

主流程继续:
  计算 10 * 2 = 20
```

## 五、一元运算符的特殊处理

### 5.1 什么是一元运算符？

```c
--5    // 两个负号 = 正5
-3+4   // 负号是一元运算符
+-3    // 正号也是一元运算符
```

### 5.2 为什么不用递归处理一元运算符？

**错误设计（会导致栈溢出）：**

```c
Unary ::= ('+' | '-') Unary | Primary
// 如果输入 ---5，会递归 5 次
```

**正确设计：使用循环计数**

```c
Unary ::= ('+' | '-')* Primary
// 统计有多少个负号，如果是奇数则取反
```

### 5.3 代码实现

```c
double parseUnary() {
    // 第1步：统计负号个数
    int negative_count = 0;
    while (当前是 + 或 -) {
        if (是 -) negative_count++;
        parserNextToken();  // 消费运算符
    }

    // 第2步：获取操作数（只递归一次）
    value = parsePrimary();

    // 第3步：应用负号
    if (negative_count % 2 == 1) {
        value = -value;
    }

    return value;
}
```

### 5.4 图解 `--5`

```
输入: --5

第1步：统计负号
  negative_count = 2

第2步：获取操作数
  parsePrimary() 返回 5

第3步：应用负号
  2 % 2 == 0，所以不取反
  返回 5
```

## 七、关键设计原则总结

### 7.1 递归用于：遇到括号需要重新开始分析

```c
if (是 '(') {
    parseExpression();  // 递归，新的开始
}
```

### 7.2 循环用于：处理同一优先级的连续运算符

```c
while (是 + 或 -) {
    // 处理完一个，继续检查下一个
}
```

### 7.3 迭代用于：一元运算符链

```c
while (是 + 或 -) {  // 不是递归，是循环
    count++;
}
```

### 7.4 为什么这样设计？

| 场景 | 方法 | 原因 |
|------|------|------|
| 括号 `(...)` | 递归 | 需要重新从最低优先级开始解析 |
| `+ - * /` 链 | 循环 | 同优先级左结合，顺序执行 |
| `--+` 符号 | 迭代 | 只关心最终正负，不需递归 |

## 八、与 LL(1) 文法的对应关系

### 8.1 本解析器对应的文法

```
Expression ::= Term (( '+' | '-' ) Term)*
Term       ::= Unary (( '*' | '/' ) Unary)*
Unary      ::= ('+' | '-')* Primary
Primary    ::= NUMBER | '(' Expression ')'
```

### 8.2 每个规则对应一个函数

```
Expression() ←→ 规则 Expression
Term()       ←→ 规则 Term
Unary()      ←→ 规则 Unary
Primary()    ←→ 规则 Primary
```

这就是"递归下降"的含义：**用互相递归的函数调用，自顶向下地"下降"到语法树的叶子节点**。

## 九、实际代码结构

以下是本项目解析器的函数调用关系：

```c
// 主入口
parserEvaluateExpression()
    │
    ├─ lexerInit()           // 初始化词法分析器
    ├─ parserNextToken()     // 获取第一个 token
    │
    └─ parseExpression()     // ⚡ 开始递归下降
            │
            ├─ parseTerm()
            │       │
            │       └─ parseUnary()
            │               │
            │               └─ parsePrimary()
            │                       │
            │                       ├─ 返回数字
            │                       └─ 或递归 parseExpression() 处理括号
            │
            ├─ while (是 * 或 /)
            │       ├─ parseUnary()
            │       └─ applyMul/applyDiv()
            │
            └─ while (是 + 或 -)
                    ├─ parseTerm()
                    └─ applyAdd/applySub()
```

## 结语

递归下降解析器是一种直观、高效的语法分析技术，特别适合**表达式解析**这种具有明确层次结构和优先级的场景。

核心要点：
1. **分层对应优先级**：Expression < Term < Unary < Primary
2. **递归处理括号**：遇到 `(` 就递归调用 `parseExpression()`
3. **循环处理运算符链**：同一优先级的运算符用循环顺序处理
4. **迭代处理一元运算符**：用计数器替代递归，避免栈溢出

理解递归下降的关键是：**想象函数调用形成的那棵树，从根节点"下降"到叶子节点的过程中完成解析**。
